## 2. 方案B主线优化的核心假设

### 2.1 主假设

**XAUUSD 在日线级别上，当价格偏离 20 日均线超过 1 个标准差时，存在可被系统性捕获的均值回归倾向，且该倾向在不同市场阶段（趋势/震荡）强度不同。**

当前 round23 版本未对市场状态做区分，所有均值回归信号一律入场。B 主线优化的核心方向是：**在保持执行干净的前提下，提升信号精准度和跨阶段稳定性**。

### 2.2 子假设（待逐一验证）

| 编号 | 假设 | 验证方式 |
|------|------|---------|
| H1 | 在强趋势行情中做均值回归会显著拉低 Sharpe，过滤强趋势可提升稳定性 | 加入趋势强度过滤器（如长均线斜率 / ADX）进行对比 |
| H2 | z-score 入场阈值 -1.0 过于宽松，提高至 -1.5 可减少噪声交易 | 网格搜索 z_entry ∈ {-1.0, -1.2, -1.5, -2.0} |
| H3 | 固定止盈倍数 3.0 × ATR 未充分捕捉大幅回归行情，改为追踪止损可提升盈亏比 | 对比 trailing-stop 版本与固定 TP 版本 |
| H4 | SMA 窗口 20 日对较慢的黄金走势偏短，20~50 日范围内存在更优参数 | 网格搜索 sma_period ∈ {15, 20, 30, 40, 50} |
| H5 | 4 天最大持仓天数对均值回归过短，允许更长持仓可减少过早止损 | 测试 max_hold_days ∈ {4, 6, 8, 10} |

## 3. 实施计划

### 3.1 优化路线图

```
Round 25：趋势过滤器（验证 H1）
    ↓ 以 round25 最优版本为新基线
Round 26：z-score 阈值网格（验证 H2）
    ↓
Round 27：止盈机制对比（验证 H3，trailing-stop vs 固定 TP）
    ↓
Round 28：SMA 窗口网格（验证 H4）
    ↓
Round 29：持仓天数网格（验证 H5）
    ↓
Round 30：综合最优参数整合版本
    ↓
Round 31（如需）：最终成本/滑点敏感性验证
```

### 3.2 每轮执行规范

1. 以上一轮通过验证的最优版本为 Baseline。
2. 每轮只改变一个变量（单因子对照）。
3. 必须在全样本（2023-01 ~ 2025-12）和弱窗口（2025-07 ~ 2025-12）同时运行。
4. 决策标准：变体的 Sharpe ≥ Baseline 且 MaxDD 未恶化超过 20%，方可纳入主线。

### 3.3 文件命名规范

- 代码：`xauusd_round{N}_{描述}_v{版本}.py`，存放于 `Program/B-mainline/code/`
- 报告：`{序数}-round.md`（如 `25th-round.md`），存放于 `Program/B-mainline/reports/`

---

## 4. 风险管理约束（全程保持不变）

1. 单笔风险不超过初始资金的 **1%**（当前 0.5%，可在验证后提升但不超过 1%）。
2. 最大回撤不得超过 **5%**（全样本）。
3. 弱窗口 Sharpe 不得低于 **0.5**。
4. 每轮成本敏感性：在 TradeCost=1.0 下仍需保持正 PnL。

---

## 5. 失效条件（何时停止该方向）

1. 连续 3 轮变体均未能在弱窗口超越当前 Baseline Sharpe。
2. 全样本 MaxDD 超过 5%。
3. 在趋势过滤加入后交易次数下降到全样本 < 50 次（信号过于稀疏，不具统计意义）。

## 7. 优化结束判定条件

### 7.1 正常结束（满足任意一项即可停止）

| 条件 | 说明 |
|------|------|
| **充分优化** | 连续 2 轮改动后绩效提升 < 5%（Sharpe 变化 < 0.05），且无明显待验证假设剩余 |
| **边际收益递减** | 剩余子假设（H1–H5）已全部验证，最优组合已确认，再改参数提升极小 |
| **无新改动可尝试** | 所有合理信号增强、参数网格、止盈机制均已测试，无新方向 |
| **Hard 问题阻塞** | 某个关键假设无法在 3 轮内解决，写入 `hard.md`，等待讨论后再决定是否继续 |

### 7.2 强制结束

| 条件 | 说明 |
|------|------|
| **达到轮次上限** | 本次授权 15 轮（round 25 ~ round 39），超出即停止 |
| **回撤超限** | 任一变体全样本 MaxDD > 5%，停止该方向 |
| **信号过稀疏** | 全样本 TradeCnt < 50，不具统计意义，停止 |

### 7.3 结束时的收尾工作

1. 写入 `Program/B-mainline/reports/B-mainline-optimization-closeout.md`，内容包括：最终选定版本、关键绩效指标、各假设验证结论、未解决问题列表。
2. 更新 `Program/current-mainline.md`，将主线指向 B-mainline 最优脚本。
3. 更新 `Program/Overview.md` 第 0 节状态。
